## Mental-model update

| Previous assumption | Updated mental model | Consequence |
|---|---|---|
| Human-in-the-loop means placing an approval screen after an agent acts | Human involvement should be risk-routed at the action boundary using calibrated confidence, consequence, reversibility, and policy | Design a decision router, not a universal approval queue |
| Confidence is explanatory metadata | Confidence is an operational control input whose usefulness depends on calibration and action-specific loss | Track calibration error, false-auto-apply rate, review yield, and downstream correction cost |
| Rationale is equivalent to auditability | Rationale is one provenance field; auditability also requires input evidence, policy version, actor identity, action payload, outcome, and immutable timestamps | Store a complete decision envelope |
| Approval creates a security boundary | Approval UX is not authorization enforcement | Keep server-side permissions, scoped identities, and policy enforcement independent of review surfaces |

## Architecture implications

```text
candidate action
  → evidence and rationale capture
  → action-specific confidence calibration
  → policy and consequence classification
  → auto-apply | suggest-for-review | deny
  → immutable decision envelope
  → outcome and correction telemetry
  → threshold recalibration
```

| Architecture element | Requirement |
|---|---|
| Decision envelope | Evidence references, rationale, confidence, calibration version, policy version, actor identity, proposed action, approval state, execution result, and timestamps |
| Routing policy | Thresholds must differ by action type, reversibility, blast radius, and repository risk class |
| Security boundary | Authorization and least privilege must be enforced server-side and must not depend on the approval panel |
| Evaluation | Measure expected loss, false-auto-apply rate, unnecessary-review rate, reviewer override rate, correction rate, and time-to-resolution |
| Feedback loop | Recalibrate confidence and thresholds from accepted, rejected, reverted, and corrected actions |
