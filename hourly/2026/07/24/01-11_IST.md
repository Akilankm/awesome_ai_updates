## Mental-model update

| Previous assumption | Updated mental model | Consequence |
|---|---|---|
| Performance portability means one kernel implementation runs efficiently everywhere | Performance portability means one high-level kernel intent can lower into different backend-specific implementations under measurable contracts | Separate semantic correctness from backend performance and validate both independently |
| Triton-style GPU kernels are the default center of custom ML kernel engineering | The durable abstraction is a hardware-heterogeneous kernel authoring stack with pluggable lowerings | Invest in IRs, backend interfaces, autotuning, conformance tests, and reproducible benchmark envelopes |
| A single headline throughput number establishes portability | Portability requires a performance envelope across shapes, dtypes, hardware generations, compiler versions, and numerical tolerances | Require benchmark matrices, regression thresholds, compile-time and tuning-cost accounting |
