## Why this changes the repository

| Previous assumption | Updated mental model | Architecture or evaluation consequence | Evidence |
|---|---|---|---|
| A fixed red-team dataset is an adequate release gate | Agent defenses require **adaptive adversarial evaluation** in which attacks iterate against the actual model, harness, tools, and policy boundary | Maintain attacker models, held-out environments, transfer tests, capability-preservation checks, and continuous regression suites rather than relying on static jailbreak lists | [OpenAI](https://openai.com/index/unlocking-self-improvement-gpt-red/) · [Adaptive attacks paper](https://aclanthology.org/2025.findings-naacl.395/) |
| A lower attack-success rate alone proves a safer model | Robustness must be paired with utility and over-refusal evaluation | Track attack success, benign task completion, refusal calibration, tool correctness, and irreversible-action prevention together | [OpenAI](https://openai.com/index/unlocking-self-improvement-gpt-red/) |
| Prompt injection is primarily a text-filtering problem | Prompt injection is an adversarial control-system problem spanning external content, tool permissions, action policies, and model behavior | Combine adversarial training with least privilege, action authorization, data/instruction separation, monitoring, and recovery controls | [OpenAI](https://openai.com/index/unlocking-self-improvement-gpt-red/) · [Public competition](https://arxiv.org/abs/2603.15714) |

## Engineering implications

| Signal | Engineering implication | Recommended response | Evidence |
|---|---|---|---|
| Attackers can adapt after observing model and tool behavior | Static benchmark saturation can hide residual vulnerabilities | Run iterative black-box and white-box attack generation against held-out agent environments | [OpenAI](https://openai.com/index/unlocking-self-improvement-gpt-red/) |
| Attack data can improve production robustness | Red-team generation can become a training and release feedback loop | Version attack corpora, preserve provenance, deduplicate strategies, retrain defenses, and re-evaluate on held-out scenarios | [OpenAI](https://openai.com/index/unlocking-self-improvement-gpt-red/) |
| Robustness can be faked by refusing more work | Safety evaluation must preserve useful capability | Require paired security–utility scorecards and task-completion controls | [OpenAI](https://openai.com/index/unlocking-self-improvement-gpt-red/) |
| Simulated attacks can transfer to live agent systems | Staging environments must model real tools and authorization boundaries closely | Build production-faithful security twins and require approval before testing irreversible actions | [OpenAI](https://openai.com/index/unlocking-self-improvement-gpt-red/) |
