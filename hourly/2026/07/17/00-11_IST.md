## Architecture implications

| Signal | Architecture implication | Recommended engineering response | Evidence |
|---|---|---|---|
| Dataset processing can execute attacker-controlled logic | Dataset ingestion belongs inside a hostile-input security boundary, not a trusted preprocessing tier | Disable remote code by default; sandbox parsers; use least-privilege workers, egress controls, ephemeral credentials, signed artifacts, and admission policies | [Hugging Face disclosure](https://huggingface.co/blog/security-incident-july-2026) |
| Agentic attacks generate thousands of coordinated actions | Conventional alert-by-alert investigation is insufficient for machine-speed campaigns | Preserve structured action provenance, correlate at campaign level, maintain offline-capable forensic models, and rehearse credential-rotation playbooks | [Hugging Face disclosure](https://huggingface.co/blog/security-incident-july-2026) |
| Monolithic system prompts become operational control planes | Prompt changes require software-engineering controls comparable to code | Introduce prompt modules, imports, static validation, generated artifacts, semantic diffs, unit tests, and CI release gates | [Google Developers Blog](https://developers.googleblog.com/building-scalable-ai-agents-with-modular-prompt-transpilation/) |
| Enterprise web grounding is becoming provider-pluggable | Retrieval quality, provenance, retention, and cost must be evaluated independently from the model | Define a grounding-provider interface and benchmark citation precision, coverage, latency, cost, freshness, caching, and policy compliance | [Google Developers Blog](https://developers.googleblog.com/en/expanding-choice-in-gemini-enterprise-agent-platform-introducing-grounding-with-parallel-web-search/) |

## Mental-model updates

| Previous assumption | Updated mental model | Consequence | Evidence |
|---|---|---|---|
| Model and API endpoints are the primary AI attack surface | Data-processing pipelines, dataset loaders, templates, workers, credentials, and autonomous action infrastructure are equally critical attack surfaces | Threat-model the full AI supply and execution chain | [Hugging Face disclosure](https://huggingface.co/blog/security-incident-july-2026) |
| Prompts are configuration text | At production scale, prompts are compiled control-plane artifacts with dependencies, validation, and release semantics | Apply source control, testing, build, and deployment discipline | [Google Developers Blog](https://developers.googleblog.com/building-scalable-ai-agents-with-modular-prompt-transpilation/) |
| Web grounding is a model feature | Grounding is an interchangeable infrastructure layer whose provider can be selected and evaluated separately | Build portable retrieval contracts and independent evidence-quality tests | [Google Developers Blog](https://developers.googleblog.com/en/expanding-choice-in-gemini-enterprise-agent-platform-introducing-grounding-with-parallel-web-search/) |

## Meeting language

| Situation | Credible formulation |
|---|---|
| AI-platform security | “Dataset ingestion must be treated as hostile code-adjacent input, with sandboxing, least privilege, credential isolation, and auditable provenance.” |
| Agent security operations | “Machine-speed campaigns require campaign-level action correlation and forensic tooling that remains usable when hosted-model guardrails or data-residency constraints block analysis.” |
| Prompt architecture | “Our prompts should be compiled and tested control-plane artifacts, not manually concatenated strings embedded across services.” |
| Grounding architecture | “We should decouple the grounding provider from the model and evaluate citation precision, freshness, latency, retention, and policy compliance independently.” |
