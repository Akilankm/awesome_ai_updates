## Narrative shifts

| Previous framing | Better framing | Why | Evidence |
|---|---|---|---|
| AI coding value as generated volume or activity | AI coding value as accepted, quality-gated outcomes per unit cost | Generated output can increase while reliability, maintainability, coverage, rework, or economics worsen | [GitHub Code Quality](https://github.blog/changelog/2026-07-20-github-code-quality-is-now-generally-available/) |
| AI review replacing static analysis | Layered deterministic and AI-assisted quality evidence | Deterministic and probabilistic analysis have different reproducibility, coverage, and error characteristics | [GitHub Code Quality](https://github.blog/changelog/2026-07-20-github-code-quality-is-now-generally-available/) |
| Quality dashboards as governance | Evaluate-first, ruleset-enforced repository policy | Dashboards inform; gates operationalize policy and require thresholds, exceptions, and rollback | [GitHub Code Quality](https://github.blog/changelog/2026-07-20-github-code-quality-is-now-generally-available/) |
| AI coding adoption as seats and total requests | AI coding adoption as repository-scoped engineering outcomes | Repositories differ in criticality, architecture, risk, review economics, and readiness | [GitHub repository metrics](https://github.blog/changelog/2026-07-17-repository-level-github-copilot-usage-metrics-generally-available/) |
| Safety testing as a fixed jailbreak suite | Safety testing as continuous adaptive adversarial evaluation | Static tests can saturate and miss attackers that optimize against the deployed system | [OpenAI GPT-Red](https://openai.com/index/unlocking-self-improvement-gpt-red/) |
