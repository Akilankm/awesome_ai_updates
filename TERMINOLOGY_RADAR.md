## Narrative shifts

| Previous framing | Better framing | Why | Evidence |
|---|---|---|---|
| Cyber benchmark as an isolated test utility | Cyber benchmark as an adversarial execution environment | The evaluated system may optimize against the harness, dependency path, secrets, and external trust boundaries | [OpenAI](https://openai.com/index/hugging-face-model-evaluation-security-incident/) |
| Disabling refusals as a model setting | Safety-control ablation as a governed system-state transition | Removing model controls increases reliance on identity, network, sandbox, monitoring, and abort controls | [OpenAI](https://openai.com/index/hugging-face-model-evaluation-security-incident/) |
| Package proxy as ordinary infrastructure | Package proxy as mediated egress and privilege boundary | A vulnerability in an allowed dependency path enabled broader connectivity | [OpenAI](https://openai.com/index/hugging-face-model-evaluation-security-incident/) |
| One model for the entire agent | Task-conditioned portfolio with explicit routing, fallback, and route-level evidence | Planner, throughput, multimodal, and specialized stages have different quality, latency, cost, and risk profiles | [Google](https://blog.google/innovation-and-ai/models-and-research/gemini-models/gemini-3-6-flash-3-5-flash-lite-3-5-flash-cyber/) |
| Token price as the AI cost metric | Total cost per accepted task | Reasoning length, tool calls, retries, latency, failures, and human rework affect economics | [Google](https://blog.google/innovation-and-ai/models-and-research/gemini-models/gemini-3-6-flash-3-5-flash-lite-3-5-flash-cyber/) |
| Agent safety as per-action allow/deny | Agent safety as trajectory-level intent, outcome, and containment | Safe-looking steps can compose into unauthorized outcomes | [OpenAI](https://openai.com/index/safety-alignment-long-horizon-models/) |
