## Narrative shifts

| Previous framing | Better framing | Why | Evidence |
|---|---|---|---|
| Human-in-the-loop as a universal approval queue | Risk-routed intervention based on uncertainty, consequence, reversibility, and policy | Review capacity should be spent where expected error cost is highest | [GitHub](https://github.blog/changelog/2026-07-23-agent-automation-controls-in-github-issues-in-public-preview) |
| High confidence as proof of safety | Calibrated confidence as one input to an action policy | Confidence can be miscalibrated and does not encode blast radius or authorization | [GitHub](https://github.blog/changelog/2026-07-23-agent-automation-controls-in-github-issues-in-public-preview) |
| Approval as security | Approval as workflow control plus independent server-side authorization | GitHub explicitly distinguishes the two | [GitHub](https://github.blog/changelog/2026-07-23-agent-automation-controls-in-github-issues-in-public-preview) |
| One portable kernel implementation | Stable kernel intent plus backend-specific lowerings and measured contracts | Accelerators differ in programming model, memory hierarchy, compiler path, and optimal schedules | [PyTorch](https://pytorch.org/blog/helion-on-tpu-towards-hardware-heterogeneous-kernel-authoring/) |
| Agent as model plus tools | Agent as a governed workflow service with explicit lifecycle controls | Reliability depends on scope, policy, evaluation, escalation, telemetry, and controlled change | [OpenAI Presence](https://openai.com/index/introducing-openai-presence/) |
