## Narrative shifts

| Previous framing | Better framing | Why | Evidence |
|---|---|---|---|
| Mobile assistants compete mainly on model quality | Mobile assistants compete as model + tool + OS-capability + data-access systems | Effective access to platform features and governed data interfaces can determine whether a rival assistant can deliver equivalent workflows | [European Commission](https://digital-markets-act.ec.europa.eu/developer-portal/interoperability/alphabet-specification-proceedings-interoperability-ai-services_en) |
| Platform interoperability means unrestricted openness | Interoperability is capability-specific access governed by privacy, security, consent, process, and revocation | Binding access obligations still preserve trust boundaries and implementation controls | [European Commission](https://digital-markets-act.ec.europa.eu/commission-provides-guidance-google-ai-interoperability-android-and-sharing-google-search-data-under-2026-07-15_en) |
| Search-data portability means bulk data export | Search-data portability is a governed data-product interface | Eligibility, anonymisation, pricing, retention, and cyber-risk review define usable access | [European Commission](https://digital-markets-act.ec.europa.eu/commission-provides-guidance-google-ai-interoperability-android-and-sharing-google-search-data-under-2026-07-15_en) |
| Agent skills as reusable prompt packages | Agent skills as governed software supply-chain artifacts | Risks and policy decisions arise during admission, retrieval, selection, execution, update, and revocation | [SkillSec-Eval](https://arxiv.org/abs/2607.13987) |
| Safety testing as a fixed jailbreak suite | Safety testing as continuous adaptive adversarial evaluation | Static tests can saturate, overfit known attacks, and miss attackers that optimize against the actual deployed system | [OpenAI GPT-Red](https://openai.com/index/unlocking-self-improvement-gpt-red/) · [Adaptive attacks](https://aclanthology.org/2025.findings-naacl.395/) |
| AI security centered on model endpoints | AI security spans the full data, execution, credential, cluster, artifact, and reusable-capability chain | Dataset-processing and skill-lifecycle paths can introduce execution and policy risk | [Hugging Face disclosure](https://huggingface.co/blog/security-incident-july-2026) · [SkillSec-Eval](https://arxiv.org/abs/2607.13987) |
| Prompts as configuration strings | Prompts as compiled, validated control-plane artifacts | Production prompt systems have dependencies, shared policies, release risk, and regression behavior | [Google Developers Blog](https://developers.googleblog.com/building-scalable-ai-agents-with-modular-prompt-transpilation/) |
| Grounding as a model capability | Grounding as a provider-pluggable infrastructure subsystem | Provider selection affects evidence quality, freshness, retention, cost, caching, and policy | [Google Developers Blog](https://developers.googleblog.com/en/expanding-choice-in-gemini-enterprise-agent-platform-introducing-grounding-with-parallel-web-search/) |
