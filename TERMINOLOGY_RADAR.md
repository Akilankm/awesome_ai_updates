## Narrative shifts

| Previous framing | Better framing | Why | Evidence |
|---|---|---|---|
| Agent safety as per-action allow/deny | Agent safety as trajectory-level intent, outcome, and containment | Safe-looking steps can compose into unauthorized outcomes over long horizons | [OpenAI](https://openai.com/index/safety-alignment-long-horizon-models/) |
| Pre-deployment evaluation as the release decision | Evaluation plus staged, monitored, reversible deployment | Deployment reveals horizon-specific failures that fixed suites may miss | [OpenAI](https://openai.com/index/safety-alignment-long-horizon-models/) |
| Incidents as postmortems | Incidents as executable evaluation assets | Replaying failures converts operational learning into durable safeguards | [OpenAI](https://openai.com/index/safety-alignment-long-horizon-models/) |
| AI coding value as generated volume | AI coding value as accepted, quality-gated outcomes per unit cost | Generated output can rise while reliability, rework, or economics worsen | [GitHub Code Quality](https://github.blog/changelog/2026-07-20-github-code-quality-is-now-generally-available/) |
| Safety testing as a fixed jailbreak suite | Continuous adaptive adversarial evaluation | Static tests can miss attackers that optimize against the deployed system | [OpenAI GPT-Red](https://openai.com/index/unlocking-self-improvement-gpt-red/) |
